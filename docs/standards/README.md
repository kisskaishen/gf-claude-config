# 编码规范与规则

> 本文档集定义了 CSD-GFUC-Web 项目的开发规范，所有团队成员的代码提交应遵守以下规则。

## 规则列表

| 文档                                       | 适用范围         | 强制程度 |
| ------------------------------------------ | ---------------- | -------- |
| [Vue 3 组件开发约定](vue-conventions.md)   | 所有 `.vue` 文件 | ⭐ 推荐  |
| [TypeScript 编码规范](typescript-rules.md) | 所有 `.ts` 文件  | ⭐ 推荐  |
| [Git 工作流程与提交规范](git-workflow.md)  | 所有 Git 操作    | 🔴 必须  |

## 总则

1. **一致性优先**：新代码的风格应与其所在文件的现有风格一致
2. **TypeScript 优先**：禁止使用 `any` 逃避类型检查，除非有充分理由并添加注释
3. **尽可能复用**：在新建组件/函数前，先搜索项目中是否已有类似实现
