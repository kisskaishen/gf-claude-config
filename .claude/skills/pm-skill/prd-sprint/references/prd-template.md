# PRD 模板 & 写作规范

> AI 生成 PRD 时按此模板结构输出。

---

## 输出格式


```markdown
# {功能名称} PRD

> {产品名} {功能名称}功能需求文档
> 版本：v1.0 | 日期：{YYYY-MM-DD} | 状态：初稿
```

---

## 章节结构 & 写作规范

# （新）需求文档模板和指引

# **一、需求背景**

- **业务背景（可选）**：*简要介绍提出该需求所处的业务环境，包括政策环境、市场环境、竞争态势、行业趋势等。这有助于相关方理解需求提出的宏观背景。*

- **现状问题（必选）**：*详细描述当前业务或系统中存在的具体问题或不足，这些问题或不足应直接关联到需求的提出。*

- **需求目的（必选）**：*基于现状问题，阐述该需求需达成的具体目标，即希望通过这个需求解决什么问题、达到什么效果。需求目的应明确、具体。*

# 二、需求修订记录

*（必选）记录需求文档每次修订/新增的关键信息。*

- *修订日期：开始/完成需求撰写的时间；*

- *上线时间：填写上线时间；撰写需求文档时，可不填写，评审时或确认版本后，填上上线时间；*

- *修订内容：为本次修订的关键信息；本次修订内容较多，用有序号“1\.”引出，逐条简要说明修订信息*

|修订日期|上线时间|修订人|修订内容|修订理由|
|---|---|---|---|---|
|*202x\-xx\-xx*|*202x\-xx\-xx*|*xx*|1. *新增 XX*<br>2. *增加查询条件*|*本次修订背景*|

# **三、业务价值**

3\.1 业务价值

*（必选）****4****个方向，****3****个维度，总计****12****个价值点，任何一个项目或者需求，都至少要有1个价值点，且价值点需要有落地衡量跟进指标；定量指标，需做埋点，基于数据跟进落地*

- *建议优先级判断标准：*

    - *客户 \&gt; 战略贡献 \&gt; 组织内部 \&gt; 供应商/伙伴 （突出以客户为中心的导向）*

    - *定量指标 \&gt; 定性指标*

    - *同一指标的数值越大优先级越高*

    - *有多维度指标，需业务综合比较做出最优指标决策*

## **1、验收场景**

*根据需求功能点描述验收内容（多语言、多时区现在规范）*

- *多语言（必选）：需要支持多语言需求，除业务录入数据外，系统操作界面呈现的内容均需满足多语言*

- *时间格式（必选）：每个系统列表展示的时间，必需支持自定义时间格式转换；验证时，需验证时间格式是否支持自定义转换。当前支持转换的格式：*

    - H:M:S dd/mm/yyyy

    - H:M:S mm/dd/yyy

    - mm/dd/yyyy H:M:S

    - dd/mm/yyyy H:M:S

    - yyyy/mm/dd H:M:S

- *数据安全（非必选）：涉及收件人、寄件人、快递员的个人信息，需验收数据安全*

## 性能

*查询、导出的反馈时间*

- **可用率：指的是系统在一个特定时间段内能够正常运行并提供服务的比例**

    - 99\.9%:也称为\&\#34;三个九\&\#34;,意味着每年有大约8\.76小时的停机时间

    - 99\.99%:也称为\&\#34;四个九\&\#34;,意味着每年有大约52\.56分钟的停机时间

    - 99\.999%:也称为\&\#34;五个九\&\#34;,意味着每年有大约5\.26分钟的停机时间

- **响应时间：系统在执行特定操作时的最大响应时间。例如,\&\#34;用户登录操作应在1秒内完成。\&\#34;**

- **并发量：在并发用户数达到预设值时,系统仍能保持稳定运行。**
