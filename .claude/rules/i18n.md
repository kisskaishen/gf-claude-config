# 国际化（i18n）

> 仅在项目已国际化时适用（如使用 `vue-i18n`）。单语言项目完全跳过。

## 文案字符串

- 模板或脚本中无硬编码的面向用户文本 — 每个字符串通过翻译键（`t('...')`）处理。键有命名空间且稳定；不要以英文句子作为键名。
- 绝不通过拼接翻译片段来组合句子 — 各语言语序不同。改用带命名占位符的插值。
- 使用语言库的复数化功能（不用手动 `count === 1` 分支）；在语言环境需要时处理性别/语境。
- 当翻译信息可能嵌入不可信数据时，绝不将 `t()`/`$t()` 的输出用 `v-html` 渲染（已知的 i18n XSS 向量）；使用 `<i18n-t>` 组件配合 slots 处理富文本（见 `security.md`）。

## 格式化

- 使用 `Intl`（或 i18n 库的包装器）按活跃语言环境格式化日期、数字和货币 — 绝不手动格式化。
- 消息目录不放在组件中；按语言环境/路由懒加载以避免初始包体积膨胀。

## RTL 与布局

- 使用 CSS 逻辑属性（`margin-inline-start`/`-end`、`text-align: start`/`end`、`inset-inline-*`）支持 RTL，而非使用物理 left/right — 在 Tailwind 中使用 `ms-`/`me-`、`text-start`/`text-end`、`start-*`/`end-*` 工具类。从根元素的 `dir` 属性驱动方向。
- 不假设文本长度 — 布局必须容忍比英文长 30-40% 的翻译而不截断或溢出。
